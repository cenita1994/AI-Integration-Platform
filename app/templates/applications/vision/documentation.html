Cat and Dog Image Classification with Unknown Image Detection

A Computer Vision and Deep Learning application that classifies uploaded images as Cat or Dog while detecting images that do not belong to the trained categories.

The system uses two major layers: a MobileNetV2 feature similarity validation model and a MobileNetV2 CNN classification model.

Computer Vision Deep Learning Unknown Detection MobileNetV2

Project Overview

This project demonstrates a hybrid image classification system for identifying cat and dog images. Unlike a basic binary classifier, the system first checks whether the uploaded image is visually similar to the trained cat and dog dataset before allowing the CNN model to classify it.

This prevents unrelated images from being incorrectly forced into either Cat or Dog. If the uploaded image does not pass the similarity threshold, the system returns Unknown Image.

Hybrid Computer Vision Architecture

The system follows a two-layer computer vision architecture. The first layer validates whether the image belongs to the expected visual domain, while the second layer performs Cat or Dog classification.

Layer 1: Similarity Validation

MobileNetV2 is used as a feature extractor to generate image feature embeddings. These embeddings are compared with the saved similarity profile of cat and dog images.

  • Feature Extractor: MobileNetV2 ImageNet
  • Validation Method: Nearest Neighbor Similarity
  • Similarity Profile: cat_dog_similarity_profile_v4.pkl
  • Similarity Threshold: 0.50
  • Output: Known Image or Unknown Image
Layer 2: CNN Classification

If the uploaded image passes similarity validation, it is processed by the CNN model for final classification.

  • Model: MobileNetV2 Transfer Learning CNN
  • Model File: cat_dog_mobilenet_model_v2.h5
  • Classification Type: Binary Classification
  • Output Classes: Cat and Dog
  • Output: Prediction and Confidence Score

Prediction Workflow

  • 1. User uploads an image
  • 2. Image is converted to RGB
  • 3. Image is resized to 224x224 pixels
  • 4. MobileNetV2 preprocessing is applied
  • 5. Feature similarity validation is performed
  • 6. If similarity is below 0.50, result is Unknown Image
  • 7. If similarity passes, CNN classifies Cat or Dog
  • 8. Confidence score and similarity score are generated
  • 9. Gemini or local AI interpreter explains the result

Model Training Using Google Colab

The CNN model was trained in Google Colab using TensorFlow and Keras. MobileNetV2 was used as the base model for transfer learning.

Training Configuration

Unknown Image Detection

The unknown image detection feature prevents the model from forcing unrelated images into Cat or Dog. If the uploaded image is not visually similar to the trained image feature profile, the system rejects the image before CNN classification.

Decision Logic
  • If similarity score is below 0.50 → Unknown Image
  • If similarity score is 0.50 or higher → Proceed to CNN classification
  • If CNN output is below 0.5 → Cat
  • If CNN output is 0.5 or higher → Dog

AI-Based Interpretation Layer

The system provides a human-readable explanation of the prediction result. When Gemini AI is available, it generates the interpretation dynamically.

If Gemini API quota is unavailable or the request fails, the system uses a local researcher-style AI interpreter fallback. This ensures that the application still provides a professional explanation even without Gemini access.

  • Gemini AI Interpreter
  • Local Computer Vision Researcher Fallback
